# VRM Review Prompts Library

Copilot CLI（GPT-5.4）に渡すプロンプト集。レビュー種別ごとに1つ。

## 共通前提（全プロンプトの冒頭に付加）

```
あなたはAITuber配信とVRMキャラクター表現の専門家です。
添付画像を見て、技術的に具体的な改善提案を返してください。
曖昧な表現（「色が強い」「動きが固い」等）は禁止。
数値・座標・パラメータ名で指示してください。
出力は日本語、Markdown箇条書き形式。
```

---

## expression（表情レビュー）

```
このVRMキャラクターの表情をレビューしてください。

観点:
1. **blendshape の組み合わせ**: angry/happy/sad/relaxed/surprised のどれがどの強度で適用されているか推定
2. **破綻チェック**: 排他グループの衝突（例: happy + sad が同時に0.5以上）、眉毛と口の感情不一致
3. **視線と頭部の整合性**: lookAt方向が視聴者と合っているか
4. **改善提案**: 具体的な blendshape 名と強度値（0.0-1.0）で修正指示

出力フォーマット:
- 現在の状態（推定）
- 問題点（Top 3、重要度順）
- 修正案（コード例: `vrm.expressionManager.setValue('<name>', <value>)`）
```

---

## pose（ポーズレビュー）

```
このVRMキャラクターのポーズをレビューしてください。

観点:
1. **ボーン回転の自然さ**: 肩・肘・手首の可動域逸脱、不自然な捻り
2. **重心とバランス**: 片足加重時の腰位置、上半身の傾き
3. **シルエットの読みやすさ**: 配信画面で視認可能か（腕が胴体に埋もれていないか）
4. **感情との一致**: ポーズが意図する感情（元気/落胆/驚き）を伝えているか

出力フォーマット:
- ポーズ名の推定（例: rokuro / present / idle）
- ボーン単位の問題点（例: `leftShoulder.rotation.y が +15度 過多`）
- 修正案（Three.js/@pixiv/three-vrm のAPI表記で）
```

---

## lipsync（リップシンクレビュー）

```
このVRMキャラクターのリップシンクをレビューしてください。

観点:
1. **口の開き量**: 発音母音（あ/い/う/え/お）との一致
2. **FFT振幅スケール**: 音声ボリュームに対する視覚的レスポンス
3. **閉じ遅延**: 発話終了後に口が閉じるタイミング
4. **表情との干渉**: 口角（happy/sad）とリップシンクの合成破綻

出力フォーマット:
- 現在の母音推定（あ/い/う/え/お/ん）
- 問題点（特にFFT scaleとexpression scaleの衝突）
- 修正案（`lipSync.setEmotionScale(<emotion>, <value>)` 形式）
```

---

## layout（配信レイアウトレビュー）

```
この配信画面全体のレイアウトをレビューしてください。

観点:
1. **セーフゾーン**: YouTube/Twitchの上下16:9セーフエリアからの逸脱
2. **キャラ配置**: 2キャラ配信の場合、左右バランスと重なり
3. **UI要素の視認性**: チャット・通知・BGM情報の可読性
4. **ブランディング一貫性**: 配色・フォント・ロゴの統一感

出力フォーマット:
- セーフゾーン違反エリア（pxまたは%）
- 視認性Top 3 問題点
- 具体的な配置調整案（例: `キャラAを右20px, 上10px`）
```

---

## overall（総合レビュー）

```
このAITuber配信の総合的な品質をレビューしてください。

4つの観点で順に評価:
1. 表情（blendshape）
2. ポーズ（ボーン姿勢）
3. リップシンク（発話中なら）
4. レイアウト（配信全体）

各観点について:
- 現状スコア（10点満点）
- 最重要の改善ポイント1つ
- 具体的な修正コマンド（実装可能な形で）

最後に、**修正優先順位Top 5**を根拠付きで提示してください。
```
